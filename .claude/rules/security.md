# 보안 규칙

## 금지 사항
- `.env` 파일 커밋 금지
- API 키, 토큰, 비밀번호를 소스코드에 하드코딩 금지
- `unsafe` 블록은 필요성 문서화 필수

## 에러 메시지
- 내부 경로, 스택 트레이스를 사용자에게 노출하지 않음
- 에러 로깅은 `log` 크레이트 사용, 민감 정보 필터링

## 입력 검증
- 외부 입력 (CLI args, TOML 설정, PTY 출력)은 항상 검증
- 경로 입력은 트래버설 공격 방지 (canonicalize 등)

## 의존성
- `cargo audit` 정기 실행
- 최소 권한 원칙: 필요한 크레이트만 의존
